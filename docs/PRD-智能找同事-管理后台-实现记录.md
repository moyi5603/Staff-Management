## 5. 数据模型（前端类型）

与原始 PRD **§6.1 ER 概要** 对齐，TypeScript 定义见 `src/types/index.ts`：

- **Employee**：工号、姓名、花名、性别、手机/尾号、部门、岗位、工作地点（省/市/区）、出生日期、试用期截止、籍贯、政治面貌、个人介绍、微信绑定、最近登录、账号状态、技能、证书、荣誉、项目、兴趣、在职状态
- **Department**：树形 `parentId`、`leaderId`（部门负责人，兼作部门联系人）、`email`（部门邮箱）、`culture`（部门文化）、`functionDetail`（职能详细介绍）、`kpis[]`（绩效指标：名称/目标值/衡量周期）

<span style="color: #F53F3F;">**产品说明：** 不需要单独维护「部门联系人」字段（含原 PRD 中的 HRBP、行政接口人等角色联系人）。对外联络、找人咨询、AI 查询匹配时，统一使用「部门负责人」作为部门联系人；负责人手机号/邮箱随员工档案获取。</span>
- **Position**：核心职责、KPI 数量
- **Project**：成员角色（负责人/核心/一般）、状态（未启动 / 进行中 / 已结束）
- **DutyRecord**：日期、`startHour` / `endHour`（0–23 整点，结束为开区间，可跨天）、值班人
- **CatalogCategory**（证书/技能/兴趣共用）：`id`、`name`、`groups[]` → `{ title, items[] }`；标签库由 `TagCatalogContext` / `CertificateCatalogContext` 内存维护，与员工选择弹窗同源

## 6. 核心交互实现说明

### 6.1 员工列表（Story 1 对齐）

- 左侧：与部门管理同源的组织树（搜索部门名、展开/收起），点击节点筛选该部门及子部门员工；「全部员工」取消部门筛选
- 筛选区（与产品设计图一致，点击「搜索」生效）：员工姓名/工号/邮箱/电话 + 关键词、在职状态、账号状态、创建时间区间、生日区间、司龄；「重置」清空条件；左侧部门树筛选仍即时生效
- 员工字段：`accountStatus`、`createdAt`、`birthday`、`probationEndDate`、`wechatBound`、`lastLoginAt`、`workLocationProvince/City/District`；司龄按 `joinDate` 计算
- 列表列（与设计图一致）：员工姓名、工号、员工生日、入职时间、部门、工作地点、手机号码、邮箱、试用截止、是否绑定微信小程序、最近登录时间、在职状态、账号状态（行内开关）、操作（修改 / 更多：查看详情、删除）
- **导出**：按当前筛选结果导出 CSV（列与列表一致，不含岗位）
- **删除**：行内删除 / 批量删除（确认弹窗）
- **批量操作**：批量改部门、批量改状态
- 导入弹窗：三步向导；模板下载/上传为演示占位
- 分页：每页 20 条
- 列表/表单/详情共享 `EmployeeContext` 内存状态（刷新恢复 Mock）

### 6.1.1 员工表单

- 新增/编辑基础字段校验与保存；基础信息含花名、性别、出生日期、试用期截止、籍贯、政治面貌、个人介绍（选填）及工作地点（单控件省 / 市 / 区级联选择 `WorkLocationPicker`）；保存后跳转详情
- 个人照片为演示占位；个人技能、个人证书、兴趣爱好支持列表增删改（`EmployeeSkillsSection`、`EmployeeCertificatesSection`、`EmployeeInterestsSection`）

### 6.2 员工详情（Story 3 对齐）

- Tab：基础信息 / 个人证书 / 个人技能 / 项目经验 / 个人荣誉 / 兴趣爱好 / 操作日志
- 手机号脱敏展示
- 个人技能：分类标签选择弹窗（`SkillPickerModal` + `TagCatalogContext.skillCatalog`，左分类右标签云，仿 BOSS 专业技能）；数据与「技能标签管理」页同源；自定义仅写入员工档案；最多 10 项
- 个人证书：添加采用分类标签选择弹窗（数据来自 `CertificateCatalogContext`，与「证书管理」页同步，仿 BOSS 资格证书选择）；卡片上「编辑」补充发证/到期日期；最多 30 本
- 兴趣爱好：分类标签选择弹窗（`InterestPickerModal` + `TagCatalogContext.interestCatalog`）；数据与「兴趣标签管理」页同源；自定义兴趣仅保存在员工档案；最多 10 项；兴趣小组为 Mock 只读展示

### 6.3 部门管理（Story 2 部分对齐）

<span style="color: #F53F3F;">**产品说明：** 本期不实现「部门联系人」独立字段。部门详情仅展示「部门负责人」，业务与后续 AI 查询层均将负责人视为部门联系人使用。</span>

- 左侧组织树 + 右侧详情
- 详情字段：部门名称、上级部门、**部门负责人**（= 部门联系人）、部门邮箱、简介、文化、职能、绩效指标
- 已明确不做：HRBP、行政接口人、部门联系人等多角色联系人字段
- 选中态：蓝色左边框 + `#E8F0FF` 背景
- 删除约束：文案与设计一致（需先转移员工）

### 6.4 证书管理

- 路由：`/certificate/list`；菜单：企业管理 → **证书管理**
- 维护证书**分类**（左侧导航）、**分组**与**证书名称**（右侧标签云）；支持搜索
- 内存状态 `CertificateCatalogProvider`，与员工「添加证书」弹窗实时同步（刷新页面恢复 `certificateCatalog` 初始 Mock）
- 操作：新增/编辑/删除分类；新增/编辑/删除分组；添加/删除证书名称

### 6.4.1 技能 / 兴趣标签管理

- 路由：`/tag/skill`、`/tag/interest`；菜单：标签中心 → **技能标签** / **兴趣标签**
- 结构与证书管理一致：**分类**（左侧）→ **分组** → **标签名称**（右侧标签云）；支持搜索
- 内存状态 `TagCatalogProvider`（`skillTagCatalog.ts` / `interestTagCatalog.ts` 初始数据），与员工「添加技能」「添加兴趣」弹窗实时同步
- 员工侧自定义技能/兴趣**不**写入标签库，仅保存在员工档案；弹窗内回车添加时提示「不入标签库」

### 6.5 岗位管理

- 列表：岗位名称、所属部门、员工人数、KPI 数量
- **+ 新增岗位**：弹窗表单（岗位名称、所属部门、核心职责、职责详细介绍、绩效指标）
- **详情**：只读弹窗展示核心职责、详细介绍与 KPI 表格；可跳转编辑
- **编辑**：同新增表单，保存后更新列表与 Mock 内存状态（本期不接后端）
- 绩效指标行支持增删；`kpiCount` 与有效 KPI 条数同步

### 6.6 项目管理（Story 2 对齐）

- 单页「项目列表」；侧栏仅保留一个入口（已移除「项目归档」）
- 顶部 Tab：全部 / 未启动 / 进行中 / 已结束（含数量角标）
- 搜索：项目名称、主负责部门、负责人、参与人（分字段输入，点击「搜索」生效）
- **项目级别**：公司级 / 部门级 / 团队级（列表卡片徽标、详情、新增/编辑表单，默认部门级）
- **项目优先级**：高 / 中 / 低（同上，默认中）
- **新增/编辑**：弹窗表单（级别、优先级、主责部门、负责人、状态、日期、简介、参与部门、成员与角色）
- **详情**：只读弹窗；可启动/结束/编辑/删除
- **状态流转**：未启动 → 进行中（启动）；进行中 → 已结束（自动填结束日期）
- **删除**：确认后移除项目，并同步员工详情「项目经验」引用（`ProjectContext` + `EmployeeContext`）
- 卡片展示负责人、成员头像组；`EmployeeProvider` 内嵌 `ProjectProvider`

### 6.7 值班管理（Story 4 对齐）

- 侧栏仅保留「值班日历」入口（值班数据以周历呈现，不单独做列表页）
- 周历：按周展示（周一～周日），支持上一周/下一周、「本周」（跳回当前自然周）
- 筛选：部门、「覆盖小时」（显示该整点有人在班的排班）
- **+ 新增值班** / 空格「+ 添加值班」：弹窗选日期、部门、值班人、开始/结束小时（0–23）、备注；**允许多条时间重叠**
- **编辑**：修改排班或删除；**批量排班**：按日期区间为每天生成相同时间段的一条；模板下载/Excel 上传为演示占位
- 时间展示：`09:00–18:00`；跨天为 `22:00–次日06:00`
- 数据：`useState` 内存维护，刷新恢复 Mock 初始值
